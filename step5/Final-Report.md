Candidate-007

* Version 1:   
  * This Candidate started off with a simplistic output where all functionality had been limited to two functions that handled all of the slot machine’s aspects.   
* Version 2:   
  * With the given prompt, massive changes were implemented to this candidate. The code added additional multipliers based on which match was made, added a way to make bets on how much to spin for between 10, 20 & 50, and added a reset game function into the output.   
* Version 3:   
  * This version of the candidate changed the way that the output makes bets, instead allowing for a more incremental increase in bet amounts. In terms of thematic critiques. This version also added in small comments for each method made to explain their base functionality.   
* Version 4:   
  * The massive changes made to the way the winning results are handled, as instead of different sets having multipliers that inflate the winnings, the sets now correlate to what amount of tokens are won, helping in letting the game be loseable. Additionally, in terms of visuals Gemini changed the look of the output to be more neon and “3D” in an attempt to make the slot machine more visually appealing.   
* Final Version:   
  * The final version of this candidate is almost identical to version 4 of this candidate. It still maintains a betting feature where one is able to bet an amount they wish to bet on a given spin. Each pair increases the token count by 25 tokens, while every set of matches increases the token amount based on which set was made. Each spin also sends a comical line to critique AI. In the event that a user runs out of tokens, they are able to reset the game instead of restarting the application. In terms of visuals, the application is identical to Version 4, but with a more muted color scheme.   
* What Stayed Consistent?   
  * Spin animation: The way the spin animation is handled, and the base way that the function knows to check for a win result has stayed relatively consistent between all 5 runs.   
  * Checking for a win: The method candidate chose to check if all results are the same, or if there is a pair that exists in the spin result that has stayed relatively the same as well, with the only bigger changes being the change of how win amounts are added to the user’s token amounts.
